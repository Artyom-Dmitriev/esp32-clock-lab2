# Как запустить проект в Wokwi (без железа)

Wokwi — онлайн-симулятор ESP32 прямо в браузере. Wi-Fi, веб-сервер, кнопка,
светодиод, зуммер — всё симулируется. Идеально для лабы без платы.

В этой папке лежит **готовый комплект файлов** для копирования в Wokwi.

---

## Что в этой папке

| Файл | Куда вставлять в Wokwi |
|------|------------------------|
| `sketch.ino` | вкладка `sketch.ino` (главный файл прошивки) |
| `web_page.h` | новый файл, который надо создать в Wokwi |
| `diagram.json` | вкладка `diagram.json` (виртуальная схема) |
| `libraries.txt` | вкладка `libraries.txt` (пустая, библиотеки встроенные) |

---

## Способ 1: пошагово через веб-Wokwi (рекомендую)

### Шаг 1. Открыть пустой проект ESP32

Перейди в браузере: **https://wokwi.com/projects/new/esp32**

Слева увидишь редактор кода, справа — виртуальную плату ESP32.

### Шаг 2. Заменить `sketch.ino`

1. Открой вкладку `sketch.ino` (она открыта по умолчанию).
2. Выдели всё содержимое: `Ctrl+A` → `Delete`.
3. Открой файл `sketch.ino` из этой папки в любом редакторе (например, Блокнот).
4. Скопируй всё (`Ctrl+A` → `Ctrl+C`).
5. Вставь в Wokwi (`Ctrl+V`).

### Шаг 3. Создать второй файл `web_page.h`

1. Над редактором кода Wokwi есть **кнопка "+"** (рядом с вкладками `sketch.ino` / `diagram.json`).
2. Нажми её → **"New File"**.
3. В открывшемся диалоге введи имя: **`web_page.h`** (точно с маленькой буквы и подчёркиванием).
4. Жми **Create**.
5. Откроется пустой файл — скопируй в него содержимое нашего `web_page.h`.

> ⚠️ Важно: имя должно быть **точно** `web_page.h`. Иначе компилятор не найдёт
> файл при `#include "web_page.h"` в основном коде.

### Шаг 4. Заменить `diagram.json`

1. Перейди на вкладку `diagram.json` (рядом с `sketch.ino`).
2. Выдели всё → удали → вставь содержимое нашего `diagram.json`.

После этого справа должна появиться схема с компонентами:
- ESP32 (зелёная плата)
- Красная кнопка слева (`RESET`)
- Синий светодиод справа (`ALARM`)
- Резистор 330 Ом
- Зуммер (`buzzer`)

### Шаг 5. Запустить симуляцию

Нажми зелёную кнопку **▶ Play** (вверху слева симулятора).

Через 2-3 секунды в Serial Monitor (нижняя панель) появится:

```
==================================================
  ESP32 Аналоговые часы с веб-интерфейсом
  Лабораторная работа №2, вариант 5
==================================================
[NVS] Восстановлено время: 00:00:00
[WIFI] Точка доступа: ESP32_Clock / 12345678
[WIFI] IP адрес: http://192.168.4.1
[MDNS] Доступно по http://esp32.local
[HTTP] Сервер запущен на порту 80
==================================================
[LOG] Система запущена
[TIME] 00:00:01  |  uptime: 1234 ms
```

И ты услышишь короткий **"пик-пик"** зуммера + увидишь вспышку синего LED — это
стартовая индикация.

### Шаг 6. Открыть веб-страницу в виртуальном браузере

Wokwi имеет встроенный браузер для подключения к виртуальному Wi-Fi:

- На картинке ESP32 справа есть значок Wi-Fi 📶 — нажми на него.
- Откроется панель с надписью **"ESP32_Clock"** — это имитация нашей точки доступа.
- Жми **"Open browser"** → введи `192.168.4.1` → Enter.
- Откроется наша страница с аналоговыми часами!

> Если иконки Wi-Fi не видишь — у разных версий Wokwi UI может отличаться.
> Альтернатива: над панелью симуляции есть вкладки. Найди "Browser" / "Network",
> там будет адрес для подключения.

## Способ 2: сохранить проект и поделиться ссылкой

После того как всё запустилось:

1. Нажми **Save** вверху Wokwi.
2. Если ты не залогинен — Wokwi попросит регистрацию (бесплатно, через Google/GitHub).
3. После сохранения у проекта будет URL вида `https://wokwi.com/projects/123456789`.
4. Эту ссылку можно отправить преподавателю — открывается у любого без установки.

**Это идеальный способ сдачи лабы дистанционно или при плохом интернете в вузе:**
открой ссылку дома, оставь вкладку — симуляция продолжит работать в облаке.

## Что показывать на демонстрации

В Wokwi у тебя одновременно видны **все** требуемые компоненты лабы:

| Где | Что показывает |
|-----|----------------|
| **Serial Monitor** (внизу) | вывод `millis()` каждую секунду — `[TIME] HH:MM:SS \| uptime: N ms` |
| **Виртуальный браузер** | страница с аналоговым циферблатом, формами, журналом событий |
| **Виртуальная плата** | синий LED моргает при будильнике/таймере |
| **Зуммер** | пищит при будильнике/таймере (Wokwi реально проигрывает звук!) |
| **Красная кнопка** | физический сброс времени (короткое нажатие) или диагностика (длинное >2 сек) |

### Сценарий 5-минутной демонстрации

1. **(30 сек)** Показать стартовый Serial-лог и часы тикают на странице.
2. **(20 сек)** Установить время `15:30:00` через форму — стрелки прыгнут.
3. **(20 сек)** Показать "Аптайм: NN мс" — это пункт задания "миллисекунды от запуска".
4. **(20 сек)** Нажать виртуальную кнопку → время сбросилось → запись в Serial и в логе.
5. **(20 сек)** Кнопка "Сбросить" в веб-интерфейсе — то же самое.
6. **(60 сек)** Запустить таймер на 5 секунд → услышать сирену + увидеть мигание LED.
7. **(60 сек)** Поставить будильник на минуту вперёд → подождать → услышать сигнал.
8. **(20 сек)** Открыть в браузере `/api/time` — показать JSON.
9. **(20 сек)** Сжать окно браузера или открыть с телефона — показать адаптивную вёрстку.
10. **(20 сек)** Перезапустить симуляцию (▶) → время и будильник восстановились из NVS.

## Возможные проблемы

| Симптом | Решение |
|---------|---------|
| Compilation error: `web_page.h: No such file` | Шаг 3 — забыл создать файл или ошибся в имени |
| `ESPmDNS.h: No such file` | На Wokwi должно быть встроено. Если нет — удали `#include <ESPmDNS.h>` и блок `MDNS.begin()` в `setup()` (это просто отключит mDNS, остальное работает) |
| Серверный код не запускается | Подожди 3-5 секунд — `WiFi.softAP()` стартует не мгновенно |
| Звука зуммера нет | Wokwi требует разрешения на воспроизведение звука в браузере. При первом запуске в углу появится `🔊 Allow audio?` — разрешить |
| Симуляция останавливается через ~30 сек | Это лимит бесплатного тарифа Wokwi. Просто нажми ▶ ещё раз |
| Стрелки на часах не двигаются плавно | Это нормально — секундная стрелка тикает по секунде (не интерполируется), часовая и минутная плавно. По требованию задания достаточно "точности до секунды". |
| `192.168.4.1` не открывается во встроенном браузере Wokwi | Введи именно `http://192.168.4.1` со схемой |

## Лимиты бесплатного Wokwi

- Время одной симуляции: ~30-60 секунд (потом надо нажать ▶ ещё раз).
- Размер прошивки: до 4 МБ (нам надо ~800 КБ — влезает).
- WiFi softAP: поддерживается полностью.
- mDNS: поддерживается.
- Зуммер: воспроизводит звук через тег `<audio>` в браузере.
- Сохранение проекта: нужен бесплатный аккаунт.

Для сдачи лабы этого хватает с лихвой.
