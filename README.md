Дословный текст задания:

> Выводите в монитор порта текущее время с помощью функции `millis()`.

## 2. Архитектура решения — общая идея

```
┌──────────────────────────────┐         ┌────────────────────────┐
│       Телефон/ноутбук         │         │       ESP32           │
│  ┌────────────────────────┐  │         │ ┌──────────────────┐  │
│  │   Браузер              │◄─┼─Wi-Fi──►│ │   WiFi softAP    │  │
│  │   http://192.168.4.1   │  │         │ │   192.168.4.1    │  │
│  └─────────┬──────────────┘  │         │ └─────────┬────────┘  │
│            │ AJAX-запросы    │         │           │           │
│            │ /api/time, ...  │         │ ┌─────────▼────────┐  │
└────────────┼─────────────────┘         │ │   WebServer :80  │  │
             │                            │ └─────────┬────────┘  │
             │                            │           │           │
                                          │ ┌─────────▼────────┐  │
                                          │ │  Логика часов    │◄─── millis()
                                          │ │  (main.cpp)      │  │
                                          │ └─┬──────┬──────┬──┘  │
                                          │   │      │      │     │
                                          │ ┌─▼─┐ ┌─▼──┐ ┌─▼──┐  │
                                          │ │NVS│ │BTN │ │LED+│  │
                                          │ │   │ │GP0 │ │BUZZ│  │
                                          │ └───┘ └────┘ └────┘  │
                                          └────────────────────────┘
```

- **ESP32 = собственный Wi-Fi роутер** (режим softAP). Никакая внешняя сеть не нужна.
- **HTTP-сервер на порту 80** обслуживает HTML-страницу и JSON API.
- **Браузер** опрашивает `/api/time` каждую секунду через AJAX и обновляет циферблат.
- **NVS (Non-Volatile Storage)** = часть flash, где сохраняются время и будильник между перезагрузками.
- **GPIO 0** — кнопка BOOT (есть на плате) — аппаратный сброс времени.
- **GPIO 2** — встроенный синий LED — индикация будильника.
- **GPIO 25** — пьезо-зуммер (в Wokwi симулируется виртуальным buzzer'ом).

## 3. Устройство ESP32 — минимум для понимания

Если ты с микроконтроллерами впервые, вот ключевые понятия:

| Понятие | Что это |
|---------|---------|
| **MCU (микроконтроллер)** | Маленький компьютер на одном чипе: процессор + память + периферия. ESP32 — это MCU. |
| **Flash** (4 МБ у NodeMCU-32S) | Постоянная память. Сюда заливается прошивка `.bin`. Не теряется при выключении. |
| **SRAM** (520 КБ) | Оперативная память. Здесь живут переменные во время работы. Очищается при ресете. |
| **NVS** | Специальный кусочек flash для key-value настроек. Аналог `localStorage` в браузере. |
| **GPIO (General Purpose I/O)** | Ножки чипа, которыми можно управлять программно: подавать `HIGH`/`LOW` или читать. |
| **`PROGMEM`** | Атрибут для размещения константы во flash, а не в RAM. Так мы храним HTML. |
| **`setup()` / `loop()`** | Аналог `main()`. `setup()` — один раз при старте. `loop()` — крутится бесконечно. |
| **`millis()`** | Функция, возвращающая количество миллисекунд с момента включения. Тип `unsigned long` (32 бита, переполняется через ~50 дней). |

### Почему мы не пишем "просто на C", а используем Arduino?

Arduino — это набор библиотек и упрощённых функций (`pinMode`, `digitalWrite`,
`delay`, `Serial.println`), который скрывает низкоуровневые регистры процессора.
Можно писать на чистом ESP-IDF (родной SDK от Espressif), но Arduino быстрее
и проще для лаб. Производительность та же.

## 6. Как работает счётчик времени через `millis()`

`millis()` возвращает количество миллисекунд от момента включения ESP32. Тип
возврата — `unsigned long` (32 бита), значит счётчик переполняется через
**~49.7 дней** непрерывной работы.

Чтобы не таскать с собой 86400 секунд "вручную", применяется простой подход:

```cpp
unsigned long syncMillis  = 0;   // millis() в момент "синхронизации"
long          baseSeconds = 0;   // время в этот момент (сек от полуночи)

long currentSeconds() {
  unsigned long elapsedSec = (millis() - syncMillis) / 1000UL;
  return (baseSeconds + elapsedSec) % 86400L;
}
```

**Идея:** запоминаем "точку отсчёта". Сейчас (`millis()`) часы показывают
`baseSeconds + (millis() - syncMillis) / 1000`.

Когда пользователь устанавливает время или жмёт сброс, мы обновляем
`baseSeconds` и `syncMillis`. Между этими событиями ничего пересчитывать не надо
— показания всегда вычисляются из текущего `millis()`. **Ошибка не накапливается.**

Переполнение `millis()` через 49 дней корректно обрабатывается арифметикой
unsigned: `millis() - syncMillis` даёт правильный результат даже после wrap-around.

## 7. Веб-интерфейс — что видит пользователь

Адрес страницы: `http://192.168.4.1` или `http://esp32.local`.

### Карточки страницы (сверху вниз)

1. **Циферблат** (SVG): часовая, минутная, секундная стрелки. Под ним —
   цифровое время и аптайм в миллисекундах. Внизу два бэйджика: статус
   будильника и статус связи с ESP32.
2. **Установить время**: поле `<input type="time" step="1">` (выбор HH:MM:SS)
   и кнопка "Установить".
3. **Будильник**: поле выбора времени, кнопки "Включить" / "Выключить".
4. **Таймер**: большой счётчик MM:SS, поле для ввода числа секунд, кнопки
   "Запустить" / "Стоп".
5. **Сброс**: красная кнопка → подтверждение → сброс времени в 00:00:00.
6. **Журнал событий**: последние 10 событий с временной меткой (от старта).
   Обновляется раз в 5 секунд.

### Как работает обновление

Браузер не перезагружает страницу. Каждую секунду JS делает `fetch('/api/time')`
и получает JSON `{h, m, s, ms, alarm, timer, alarmH, alarmM, timerFired}`.
Затем обновляет:
- цифровое время и аптайм (текстом),
- угол поворота стрелок (через CSS `transform: rotate(...)`),
- видимость уведомлений будильника и таймера,
- бэйдж "Будильник".

Лог обновляется реже — раз в 5 секунд через `fetch('/api/log')`.

## 8. HTTP API (для AJAX и curl)

Все эндпоинты — обычные `GET`. Можно вызывать как из браузера, так и из терминала.

| Метод | URL | Параметры | Описание | Ответ |
|-------|-----|-----------|----------|-------|
| GET | `/` | — | HTML-страница | `<!DOCTYPE html>...` |
| GET | `/api/time` | — | Текущее время в JSON | `{"h":12,"m":34,"s":56,"ms":12345,"alarm":false,"timer":-1,"alarmH":-1,"alarmM":0,"timerFired":false}` |
| GET | `/api/log` | — | Журнал событий | `[{"t":1234,"m":"Время сброшено..."}]` |
| GET | `/set` | `time=HH:MM:SS` | Установить время | `OK` |
| GET | `/reset` | — | Сброс на 00:00:00 | `OK` |
| GET | `/alarm` | `time=HH:MM` или `time=off` | Будильник | `OK` / `OFF` |
| GET | `/timer` | `seconds=N` (0 = стоп) | Таймер | `OK` / `STOPPED` |

Примеры из терминала:

```bash
curl http://192.168.4.1/api/time
curl "http://192.168.4.1/set?time=18:30:00"
curl "http://192.168.4.1/alarm?time=07:00"
curl "http://192.168.4.1/timer?seconds=10"
curl http://192.168.4.1/reset
```

## 9. Дополнительные фичи — детали

### Д2. NVS-сохранение

Используем библиотеку **`Preferences`** — обёртка над NVS API ESP-IDF.

```cpp
prefs.begin("clock", false);          // открыть namespace для записи
prefs.putLong("base", baseSeconds);    // сохранить
prefs.putULong("syncOffset", millis() - syncMillis);
prefs.end();                           // закрыть
```

При следующем включении `loadFromNVS()` читает эти значения. Если ключа нет
(первый запуск), возвращается значение по умолчанию (0).

**Важно:** NVS не заменяет батарейку RTC. Пока ESP32 был выключен, время
"стоит на паузе". При включении часы продолжат с того момента, на котором
остановились. Для настоящих часов реального времени нужен модуль RTC
(например, DS3231) с батарейкой — но это уже выходит за рамки лабы.

### Д3. mDNS

`MDNS.begin("esp32")` объявляет имя в локальной сети по протоколу
**Multicast DNS** (RFC 6762). После этого устройство откликается на
`esp32.local` на любом устройстве, где есть Bonjour/Avahi (это все
современные iOS, Android, macOS и Windows 10+).

### Д4. Будильник

Логика:
- В `loop()` мы каждую итерацию проверяем: совпадают ли текущие H:M:S с
  заданным `alarmHour:alarmMinute:00`.
- Если да и `alarmTriggered` ещё не выставлен — выставляем флаг и засекаем
  `alarmStartMillis`.
- Пока флаг выставлен (60 секунд) — мигаем LED и пищим зуммером
  (прерывисто 200/200 мс на частоте 2.5 кГц).
- По истечении 60 сек — сбрасываем флаг автоматически.

### Д5. Таймер обратного отсчёта

Простой счётчик `timerSecondsLeft`. В `loop()` каждые 1000 мс уменьшаем на 1.
Когда дошёл до 0 — выставляем флаг `timerFired` и звучим "сирену" 5 секунд
(чередуем 1.5 кГц и 2.2 кГц каждые 300 мс — характерный звук).

### Д6. Лог событий

Кольцевой буфер на 10 элементов:

```cpp
struct LogEntry {
  unsigned long timestamp;
  char message[48];
};
LogEntry eventLog[LOG_SIZE];
int logHead = 0, logCount = 0;
```

Когда массив заполнен, новые записи перезаписывают самые старые. Это
идиоматичный подход для микроконтроллеров — не используем `malloc`, не
фрагментируем кучу.

### Д7. Зуммер

`tone(BUZZER_PIN, freqHz)` использует встроенный в ESP32 LEDC (LED Controller),
аппаратный PWM. Генерирует прямоугольную волну заданной частоты. На пьезо
пищалке прямоугольник на 1-3 кГц звучит как чёткий тон.

### Д11. Длинное нажатие

В обработчике кнопки сравниваем длительность удержания с `LONG_PRESS_MS = 2000`.
Если короче — обычный сброс. Если дольше — печатаем диагностику в Serial:

```
[BTN] Длинное нажатие — диагностика:
       Heap free: 250516 bytes
       Uptime:    1234567 ms
       Clients:   1 connected
```

Полезно при отладке — можно проверить, не "течёт" ли память и сколько
клиентов подключено к точке доступа.

## 11. Использование памяти

| Тип | Размер чипа | Использовано прошивкой | % |

### 14.0. Пошаговая инструкция «прошить с нуля» (10 шагов)

Если у тебя есть плата и просто нужна **готовая последовательность кликов
без выбора вариантов** — делай ровно как написано.

#### 🔧 Подготовка (один раз, дома)

1. **Установи драйвер CP210x.**
   Открой папку `Лабораторная работа 2 ПиАСИФинал/CP210x_Universal_Windows_Driver/`,
   запусти **`CP210xVCPInstaller_x64.exe`** → Next → Next → Finish.
   После этого Windows будет видеть USB-порт ESP32.

2. **Открой VS Code с проектом.**
   `File → Open Folder → выбери папку Led_NodeMCU` (где лежит `platformio.ini`).
   Подожди 10-30 секунд — внизу должна появиться синяя панель с иконками PlatformIO
   (🏠 ✓ → 🚮).

3. **Собери прошивку (если ещё не собрана).**
   Нижняя панель → нажми **✓** (галочка = Build) → подожди `[SUCCESS]` в терминале.
   Это создаст файл `Led_NodeMCU/.pio/build/nodemcu-32s/firmware.bin`.

#### 🔌 Подключение (каждый раз)

4. **Подключи плату USB-кабелем к компьютеру.**
   ⚠️ Важно: кабель должен поддерживать **передачу данных**, а не только
   зарядку. Если в Диспетчере устройств не появляется новый COM-порт —
   проблема в кабеле.

5. **Проверь что Windows видит плату.**
   Win+X → "Диспетчер устройств" → раздел **"Порты (COM и LPT)"** → должна
   появиться строка вида `Silicon Labs CP210x USB to UART Bridge (COM3)`.
   Запомни номер порта (COM3, COM4, COM5 — у всех разный).

#### ⚡ Прошивка

6. **Жми Upload в VS Code.**
   В нижней синей панели → иконка **→** (стрелка вправо) = Upload.
   Или клавиатурой: `Ctrl+Alt+U`.

7. **Подожди ~5-10 секунд.**
   В терминале пойдут строки:
   ```
   Auto-detected Flash size: 4MB
   Connecting......_____....._____
   Chip is ESP32-D0WD-V3 (revision v3.0)
   Writing at 0x00010000... (8 %)
   ...
   Hash of data verified.
   Leaving...
   Hard resetting via RTS pin...
   ```
   Если всё прошло — увидишь **`========= [SUCCESS] =========`**. Готово.

   ⚠️ **Если ошибка** `Failed to connect to ESP32: No serial data received`:
   - Зажми и держи кнопку **BOOT** на плате.
   - Не отпуская BOOT — нажми и отпусти **EN** (RST).
   - Снова нажми Upload в VS Code.
   - Когда в терминале появится `Connecting...` — отпусти BOOT.
   - Дальше прошивка пойдёт нормально.

#### 📡 Проверка

8. **Открой Serial Monitor.**
   Нижняя синяя панель → иконка **🔌** (вилка) или `Ctrl+Alt+S`.
   Должны посыпаться строки:
   ```
   ==================================================
     ESP32 Аналоговые часы с веб-интерфейсом
   ==================================================
   [WIFI] Точка доступа: ESP32_Clock / 12345678
   [WIFI] IP адрес: http://192.168.4.1
   [TIME] 00:00:01  |  uptime: 1234 ms
   [TIME] 00:00:02  |  uptime: 2234 ms
   ```
   Если вместо букв "крякозябры" — у тебя не та скорость порта. Должно быть
   **115200 бод** (это уже прописано в `platformio.ini`, не должно быть проблем).

9. **Подключи телефон/ноутбук к Wi-Fi.**
   На телефоне → `Настройки → Wi-Fi` → найди в списке сетей **`ESP32_Clock`** →
   нажми → введи пароль **`12345678`** → Подключиться.
   Появится уведомление **"Сеть без интернета. Остаться?"** → нажми
   **"Оставить подключённым"** / **"Не переключаться"** (формулировка зависит
   от Android/iOS).

10. **Открой страницу.**
    На том же телефоне открой браузер → введи в адресной строке:
    **`http://192.168.4.1`** (или `http://esp32.local`) → Enter.
    Загрузится страница с аналоговым циферблатом — стрелки тикают, формы

После шага 10 ESP32 уже не нуждается в подключении к компьютеру — её можно
питать от любого USB-зарядника, и часы продолжат работать. Прошивка

### 15.5. Памятка по терминам (если спросят определение)

- **Микроконтроллер** — однокристальный компьютер: процессор + память + периферия.
- **`millis()`** — функция, возвращающая миллисекунды от старта (`unsigned long`, переполняется через ~50 дней).
- **GPIO** — программируемая ножка чипа, может быть входом или выходом.
- **PWM** — широтно-импульсная модуляция, цифровой сигнал имитирующий аналоговый уровень.
- **softAP / Access Point** — режим Wi-Fi, в котором устройство само создаёт сеть.
- **AJAX** — асинхронный HTTP-запрос из браузера без перезагрузки страницы.
- **mDNS** — Multicast DNS, протокол резолвинга `*.local`-имён без DNS-сервера.
- **NVS** — Non-Volatile Storage, энергонезависимое хранилище ESP32 (часть flash для настроек).
- **PROGMEM** — атрибут хранения константы во flash вместо RAM.
- **Антидребезг** — программный фильтр от ложных срабатываний механической кнопки.
- **Кольцевой буфер** — структура данных, в которой указатель записи "ходит по кругу" и затирает старые данные.

## 16. Возможные вопросы преподавателя и ответы

### Вопрос: "Почему вы используете `millis()`, а не `delay()` для отсчёта времени?"

**Ответ:** `delay()` блокирует выполнение кода на указанное время — пока он
"спит", ESP32 не может обрабатывать HTTP-запросы или опрашивать кнопку.
`millis()` неблокирующий: я просто проверяю в каждой итерации `loop()`,
прошла ли уже секунда с прошлой печати. Это стандартный паттерн "non-blocking
timing" в Arduino.

### Вопрос: "Что произойдёт через 50 дней работы — `millis()` же переполнится?"

**Ответ:** `millis()` возвращает `unsigned long` (32 бита) и действительно
переполняется через ~49.7 дней. Но в моём коде везде используется арифметика
вида `millis() - последнее_событие` — для unsigned типов вычитание корректно
работает даже после переполнения. Так что часы будут идти бесперебойно.

### Вопрос: "Зачем антидребезг кнопки, если в симуляторе дребезга нет?"

**Ответ:** На реальной плате механическая кнопка при нажатии "звенит" —
контакты замыкаются и размыкаются несколько раз за миллисекунды. Без
программного фильтра моё одно нажатие воспринималось бы как 5-10 нажатий
подряд. 50 мс — стандартное окно подавления дребезга.

### Вопрос: "Почему точка доступа, а не подключение к существующему Wi-Fi?"

**Ответ:** Чтобы устройство было автономным. Если ESP32 в режиме STA
(подключение к роутеру), нужны учётные данные сети, а в чужой сети их нет.
В режиме softAP устройство само раздаёт Wi-Fi и работает где угодно — для
лабы и для демо это удобнее. Архитектурно ничто не мешает добавить второй
режим — например, переключение через кнопку или веб-форму.

### Вопрос: "Как браузер узнаёт, что время на ESP32 поменялось?"

**Ответ:** Через AJAX-опрос (polling). JavaScript на странице каждую секунду
вызывает `fetch('/api/time')`, ESP32 отвечает JSON'ом с текущим временем,
страница обновляет цифры и углы стрелок. Альтернатива — WebSocket для
push-уведомлений от сервера, но для лабы AJAX проще и надёжнее.

### Вопрос: "Почему HTML лежит в `PROGMEM`?"

**Ответ:** RAM на ESP32 ограничена (320 КБ доступно для пользователя).
Если положить 14-килобайтную HTML-строку как обычную глобальную переменную
— она съест 14 КБ оперативки впустую. `PROGMEM` кладёт данные во flash
(там 4 МБ), и `server.send_P()` отправляет их прямо оттуда без копирования.

### Вопрос: "Что хранится в NVS и почему именно там?"

**Ответ:** В NVS хранятся `baseSeconds` (базовое время) и `alarmHour/alarmMinute`
(будильник). NVS — это специально размеченная партиция flash для key-value
пар. Я выбрал её, потому что:
1. Данные переживают перезагрузку (в отличие от RAM).
2. Не надо самому управлять адресами (в отличие от EEPROM API).
3. Wear leveling — автоматическое распределение записей по ячейкам, чтобы
   flash дольше прослужила (~100 тысяч циклов).

### Вопрос: "Почему частота зуммера именно 2 кГц?"

**Ответ:** Пьезо-пищалки имеют резонансную частоту обычно в диапазоне
1.5-3 кГц — на ней звук максимально громкий при минимальной мощности.
2 кГц — это типичный частотный отклик для маленьких пищалок, и на ней
сигнал хорошо слышен ухом (полоса наибольшей чувствительности человека —
1-4 кГц).

### Вопрос: "Какие ещё были бы способы реализовать те же часы?"

**Ответ:**
- **Через RTC-модуль** (например, DS3231 с батарейкой) — точнее, время не
  сбросится при отключении питания. Но для нашей задачи избыточно.
- **Через NTP-сервер** — синхронизация с эталонным временем из интернета.
  Но это требует выхода в интернет, что противоречит идее "автономной"
  точки доступа.
- **Через RTOS-таймер** — отдельная задача FreeRTOS, обновляющая время.
  Архитектурно красивее, но для микроконтроллера с одной "большой"
  задачей это избыточно.

### Вопрос: "А если у вас одновременно сработает будильник И таймер?"

**Ответ:** Сейчас они независимы и могут одновременно использовать LED и
зуммер — это не баг, оба будут "бороться" за пин в `loop()`. На практике
это не страшно: оба издают характерный звук, и они автоматически отключаются
через 60 и 5 секунд соответственно. Если бы потребовалось — можно сделать
приоритет (будильник важнее таймера) или явный mutex.

### Вопрос: "Как обеспечивается потокобезопасность веб-сервера и `loop()`?"

**Ответ:** В моей реализации потокобезопасность не нужна — это не
многопоточная программа. `WebServer.handleClient()` вызывается из `loop()`
в главном цикле и обрабатывает все HTTP-запросы синхронно. То есть пока
выполняется `handleSetTime()`, ничего другого в `loop()` не работает.
Это упрощает код, но накладывает ограничение: обработчик не должен быть
долгим, иначе часы "застывают".

## 17. Диагностика проблем (FAQ)

| Симптом | Причина / решение |
|---------|-------------------|
| `web_page.h: No such file or directory` (Wokwi) | Не создан второй файл. В Wokwi нажми "+" → имя `web_page.h`, вставь содержимое |
| Wi-Fi `ESP32_Clock` не виден | Подожди 5-10 секунд после загрузки. Проверь Serial — должна быть строка `[WIFI] Точка доступа: ESP32_Clock` |
| Страница не открывается на телефоне | Телефон автоматически переключился на сеть с интернетом. Зайди в настройки Wi-Fi и явно "оставить подключённым к ESP32_Clock без интернета" |
| `http://esp32.local` не работает | На Windows < 10 нет mDNS. Используй IP `192.168.4.1`. На Linux — установи `avahi-daemon` |
| После прошивки в Serial мусор | Не та скорость. В Monitor должно быть **115200 бод** |
| Не могу прошить (`Failed to connect`) | Зажми BOOT → нажми и отпусти EN/RESET → отпусти BOOT. Иногда нужно после `pio run --target upload` |
| Лог очищается при перезагрузке | Это нормально — лог хранится в RAM, не в NVS |
| Будильник не пищит в Wokwi | Проверь, что в `diagram.json` есть компонент `wokwi-buzzer` подключённый к `esp:25` |
| Время сбрасывается каждый раз | Возможно NVS не записался. Проверь Serial на наличие строки `[NVS] Восстановлено время:` |
| Длинное нажатие не работает | Удерживай кнопку **строго больше 2 секунд** перед отпусканием |

## 19. Глоссарий терминов

| Термин | Расшифровка |
|--------|-------------|
| **ESP32** | Микроконтроллер от Espressif с встроенным Wi-Fi и Bluetooth |
| **NodeMCU-32S** | Конкретная плата на основе модуля ESP-WROOM-32 (ESP32) |
| **GPIO** | General Purpose Input/Output — программируемая ножка чипа |
| **PWM** | Pulse Width Modulation — широтно-импульсная модуляция (для управления яркостью LED, оборотами моторов и т.д.) |
| **AP / softAP** | Access Point — режим Wi-Fi, в котором устройство сам становится точкой доступа |
| **AJAX** | Asynchronous JavaScript and XML — техника асинхронных HTTP-запросов из браузера без перезагрузки страницы |
| **mDNS** | Multicast DNS — протокол резолвинга имён в локальной сети без DNS-сервера (`*.local`) |
| **NVS** | Non-Volatile Storage — энергонезависимое хранилище ESP32 (часть flash для key-value пар) |
| **RTC** | Real-Time Clock — часы реального времени с независимым питанием. У ESP32 есть только "программный" RTC, без батарейки сбрасывается при отключении |
| **PROGMEM** | Атрибут размещения константы во flash вместо RAM |
| **`millis()`** | Функция Arduino, возвращающая количество миллисекунд от запуска |
| **Кольцевой буфер** | Структура данных фиксированного размера, в которой указатель записи "ходит по кругу" и затирает старые элементы |
| **Антидребезг** | Программный фильтр, отсекающий ложные срабатывания механической кнопки |
